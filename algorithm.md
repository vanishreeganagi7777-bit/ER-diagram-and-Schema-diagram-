# Algorithm

### 

### **Step1:** 

#### &#x09;**Mapping of Regular Entity Types:**

##### &#x09;			For each regular entity type create a relation(table) that includes all the simple attributes

##### &#x09;			of the regular entity. 

### **Step2:**

#### &#x09;**Mapping of Weak Entity Types:**

##### &#x09;			For each weak entity type create a relation (table) that includes all simple attributes of 				the weak entity.

### **Step3:**

#### &#x09;**Mapping of Binary Relationship Types:**

##### &#x09;			Include one side of the relationship as a foreign key in the other favor total participation.





### **Step4:**

#### &#x09;**Mapping of Binary(1-N) Relationship Types:**

##### &#x09;			For each weak entity type create a relation (table) that includes all simple attributes of 				the weak entity.





### **Step5:**

#### &#x09;**Mapping of Binary(M-N) Relationship Types:**

##### &#x09;			Create a new relation (table) who's primary key is a combination of both entities primary 				key's . Also include any relationship attributes.
